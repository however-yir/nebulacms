# Security Policy

## Supported Versions

NebulaCMS currently supports the versions listed below:

- :white_check_mark: indicates an active development roadmap, is therefore maintaining, and **will** receive Security
  Vulnerability Report.
- :x: indicates such version has already deprecated and **will not** be receiving Security Vulnerability Report.

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :x:                |
| 1.x     | :x:                |
| 2.x     | :white_check_mark: |

## Reporting a Vulnerability

We first appreciate and are very thankful that you've found a vulnerability issue in NebulaCMS. By disclosing such issue
to the NebulaCMS maintainers, you are helping NebulaCMS become safer for everyone.

To protect existing NebulaCMS users, we kindly ask you to not disclose the vulnerability to anyone except the NebulaCMS
maintainers before a fix has been rolled out.

To Report a Vulnerability, please complete the form below, and send such report by email to `hi@nebulacms.io`.

```
Vulnerability has been observed in...
  - Docker? [n/y]:
    if yes for the question above,
    - `docker -v`:
    - `docker images nebulalabs/nebulacms`:

  - by `java -jar nebulacms.jar`? [n/y]:
    if yes for the question above,
    - `uname -a`:
    - `java -version`:

- Affected NebulaCMS version(s) [e.g. v2.4.0]:
- Vulnerability self-scoring [1-10]:
- Would you like to be attributed? (Whether you agree us to appreciate you by putting your name in the CHANGELOG of the next fix release) [n/y]:
```
